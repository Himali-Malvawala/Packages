# Life.Church Content Provider

Browses the Life.Church kids curriculum library and exposes the upcoming Sunday's scheduled lessons:

- **The Loop Show** — 4th–6th grade preteen
- **Konnect HQ** — 1st–4th grade elementary
- **Crosstown** — Pre-K / Kindergarten
- **Early Childhood** — Ages 2–5

Source: <https://www.life.church/media/> (catalog) + <https://open.life.church/schedule> (weekly calendar). Free under Life.Church's "free now, free always" Open Network license.

## How it works

The provider does **not** call out to a Life.Church API at runtime. It walks a static catalog at [`data.json`](./data.json), generated by [`scripts/scrape-lifechurch.ts`](../../../scripts/scrape-lifechurch.ts).

- Each lesson stores a direct mp4 URL on the Life.Church CDN (`d.life.church/.../{slug}_720p.mp4`) — public, no auth, no signed expiry tokens, supports HTTP range.
- Scheduled lessons are pulled from the open.life.church calendar (categories 266 / 303 / 301) for the next Sunday and resolved against the catalog so the provider can surface "This Sunday" without runtime HTTP.

## Path structure

```
/                                      → "This Sunday" + 4 series
/scheduled                             → this Sunday's lessons (one per age group)
/scheduled/{scheduledId}               → single video file (leaf)
/{seriesId}                            → units within the series
/{seriesId}/{unitId}                   → lessons within the unit
/{seriesId}/{unitId}/{lessonId}        → single video file (leaf)
```

Scheduled-lesson ids are `{seriesId}-{YYYY-MM-DD}` (e.g. `loop-2026-05-10`).

## Re-running the scraper

```bash
# Full scrape (catalog + schedule, ~3 minutes)
npx tsx scripts/scrape-lifechurch.ts

# Refresh just the schedule, keep the existing catalog
npx tsx scripts/scrape-lifechurch.ts --schedule-only

# Smoke test
npx tsx scripts/scrape-lifechurch.ts --series konnect --max-per-series 5
```

The scraper:
1. Fetches `https://www.life.church/sitemap.xml` and filters URLs under `/media/{loop|konnect|crosstown|early-childhood}/`.
2. For each leaf episode page, extracts the `og:title`, `og:image`, and the canonical `d.life.church/...{slug}_720p.mp4` URL.
3. Fetches `open.life.church/schedule?category_id={id}&start_date={next-sunday}` for each kids age group, parses the calendar entry, and resolves it to a unit + lesson in the catalog. Entries that don't have a life.church/media counterpart are skipped (the two surfaces occasionally diverge; see below).
4. Writes the result to `data.json`. Rate-limited to ~250ms between requests.

## Smoke test

```bash
npx tsx scripts/test-lifechurch.ts
```

Browses the root, drills into each series + the scheduled folder, and asserts every leaf resolves to a `ContentFile`.

## Surface mismatch caveat

`open.life.church` and `life.church/media/` are two different surfaces with overlapping but non-identical content. open.life.church organizes by curriculum unit + week (e.g. "Discerning Truth - Konnect HQ, Unit 25 / Part 2"); life.church/media/ organizes by individual episode slug. The scraper bridges them by matching unit slugs, but a small percentage of scheduled entries reference content that doesn't have a public mp4 on life.church/media — those are dropped from `scheduledLessons` rather than emitting broken links.

## Player compatibility

`ContentFile.url` is a direct mp4 URL (`Content-Type: video/mp4`, supports HTTP range). Any standard mp4 player works — `react-native-video`, HTML5 `<video>`, etc.

## Attribution

Content remains the property of Life.Church. Each lesson's `providerData.sourceUrl` back-links to the canonical episode page on `life.church/media/`.
